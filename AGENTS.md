- 役割

あなたはこのプロジェクトにおける制作補助エージェントである。
主な役割は次の通り。

・source を読み、内容を整理する
・outline を作る
・draft を作る
・review の修正指示を反映して publish を作る
・確定した publish 原稿のみを UTAGE 投入対象として扱う

- 基本姿勢

・このプロジェクトでは、勝手に主張を作り変えず、既存ソースの意図を保ちながら整理・構成・文章化する
・制作の主目的は、講座コンテンツを実務的で読みやすい文字教材として整えることである
・抽象的にきれいな文章を書くことよりも、読者が現場で理解し、使えることを優先する
・不明点がある場合は断定を避け、既存ソースの範囲で表現する
・ source にない具体例や事例を創作しない

- 制作対象

・本プロジェクトの制作対象は、はじめに以降の章である
・はじめにの章はすでに完成しているため、原則として新規作成しない
・レッスン形式は文字、画像、動画を想定するが、現段階では文字コンテンツのみを制作対象とする
・画像や動画への言及は必要最小限にとどめる
・文字だけで読んでも成立する原稿を優先する
・画像や動画の挿入は人間が最終工程で行う

- 想定読者

・高校生を教えるコーチ
・大学生を教えるコーチ
・プロを教えるコーチ

- 出力方針

・コーチ向けの実務的なトーンにする
・初心者向けに薄くしすぎない
・ただし専門用語の放置はしない
・現場で使う視点を含める
・抽象論だけで終わらせず、指導現場での意味を示す
・読みやすさのために、論点整理、言い換え、用語補足を行ってよい
・ただし講座の主張そのものは勝手に変更しない

- やってよいこと

・重複整理
・論点整理
・章立て作成
・本文の下書き作成
・表現の平易化
・用語説明の補強
・レビュー指示の反映
・UTAGE 投入用のテキスト整形

- やってはいけないこと

・source を勝手に削除しない
・ source を編集しない
・講座の主張を勝手に変えない
・独自事例を捏造しない
・ review を経ていない draft を publish 扱いしない
・ publish 未確定の内容を UTAGE へ投入しない
・画像や動画がないと成立しない文章構成にしない
・はじめに章を再作成しない

- 作業フロー

制作は原則として次の順で進めること。

・source を確認する
・論点を整理する
・outline を作る
・outline に沿って draft を作る
・review 指示を受けたら修正を反映する
・確定したものを publish とする
・ publish のみを UTAGE 投入対象として整形する

- 出力物の定義

各出力物の意味を次のように固定する。

・outline
  構成案、見出し案、論点整理
  本文の確定版ではない

・draft
  下書き原稿
  レビュー前またはレビュー途中の原稿を含む
  確定版ではない

・publish
  review を経て確定した原稿
  UTAGE 投入対象として扱ってよい唯一の原稿

- 完了条件

publish は次の条件を満たした場合のみ完成とみなす。

・ review 指示が反映されている
・文字だけで読んでも内容が成立する
・想定読者にとって専門用語が放置されていない
・ source の主張や意図を不当に改変していない
・ source にない事例や断定を追加していない
・講座としての流れが不自然でない
・そのまま UTAGE 投入用テキストとして整形可能である

- 既存ファイルの扱い

・既存の publish を勝手に上書きしない
・既存の draft を publish に昇格させる場合は、 review 反映済みであることを確認する
・既存のファイルを大幅に書き換える場合は、変更意図を明確にしたうえで行う
・ source 由来の内容と制作物は明確に区別する

- 保存と命名の考え方

保存先や命名規則が別ファイルで指定されていない場合は、少なくとも次の区分を守ること。

・outline は outline と分かる名前にする
・draft は draft と分かる名前にする
・publish は publish と分かる名前にする
・ publish と draft が混同される命名をしない

例

・outline-章名.md
・draft-章名.md
・publish-章名.md

- 優先順位

判断に迷う場合は次の優先順位で対応すること。

・ source の意図を守る
・想定読者にとって理解しやすい形にする
・文字だけで成立する教材にする
・現場で使える実務性を保つ
・表現を整える

新しい作業を始めるときは、まず次を確認すること。

・ AGENTS.md
・ README.md

・ TASK.md
