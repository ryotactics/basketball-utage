# はじめに｜本講座の歩み方


## この講座で身につけてほしいこと

ここまでのセクションでは、戦術がチームづくりの中でどの位置にあるのか、そして戦略と戦術をどう区別するのかを整理してきました。

その前提を踏まえたうえで、本編に入る前に、この講座全体をどのように読み進めればよいのかを確認しておきます。

本講座は、バスケットボールの戦術をただ暗記するための講座ではありません。

もちろん、PNR、DHO、Pindown、Spain PNR、Zone、Show、Drop、Switch など、具体的な戦術やアクションは扱います。しかし本当に身につけてほしいのは、個別のプレー名を増やすことではなく、自チームに必要な戦術を選び、設計し、運用し、改善していくための考え方です。

現場では、良さそうなセットや流行しているアクションを取り入れたくなる場面が多くあります。映像で見たプレーをそのまま使いたくなることもあります。

ただし、戦術は単体で機能するものではありません。

その戦術が、

- どのような勝ち筋につながっているのか
- どの選手の強みを活かすためのものなのか
- どのアドバンテージを作り、どのアドバンテージを使うものなのか
- 相手に何を守らせ、何を諦めさせるものなのか
- 運用したあと、何を見て修正すべきなのか

ここまでつながって初めて、戦術は「覚えた知識」ではなく「現場で使える武器」になります。

本講座では、戦術をプレー集として並べるのではなく、原理 → 戦術 → 対応 → 戦略設計 → 分析改善という流れで整理していきます。


## ModuleとLessonについて

本講座では、大きな章のまとまりを MODULE と呼びます。

MODULEは、1つのテーマを深く学ぶための大きなブロックです。たとえば「オフェンス原論」「ディフェンス原論」「戦略設計・システム構築」のように、扱う領域ごとに分けています。

その中にある小さな単元を Lesson と呼びます。

Lessonは、MODULEの中で1つずつ論点を分解して学ぶための単位です。たとえば「アドバンテージとは何か」「PNRカバレッジとは何か」「映像分析をどう改善につなげるか」といった形で、現場で考えるべき問いごとに整理しています。

つまり、


MODULE = 大きな学習テーマ
Lesson = そのテーマを構成する個別の論点

という理解で読み進めてください。


## 全体の流れ

本講座は、次の順番で進みます。

## MODULE 1｜オフェンス原論

まず、オフェンスの土台になる考え方を整理します。

ここで扱う中心テーマは、アドバンテージです。オフェンスは、ただ形をなぞるものではありません。どこで優位を作り、どう使い、どのように次の優位へつなげるかが重要です。

MODULE 1では、戦術名に入る前に、オフェンスを読むための共通言語を作ります。

## MODULE 2｜オフェンス戦術大全

次に、具体的なオフェンス戦術を整理します。

オンボールスクリーン、オフボールスクリーン、ポスト、パスアクション、モーション、セットオフェンスなどを扱います。ただし、ここでも目的は「戦術名を覚えること」ではありません。

それぞれのアクションがどのようなアドバンテージを作るのか、どのロスターに合いやすいのか、どのような守られ方に対して有効なのかを考えるための骨格として読んでください。

## MODULE 3｜ディフェンス原論

オフェンスの原理を押さえたあと、ディフェンスの土台を整理します。

ディフェンスは、単に頑張るものではありません。何を守り、何を許容し、どこに相手を誘導するのかを設計する必要があります。

MODULE 3では、個人の努力だけでなく、チームとして守るための考え方を扱います。

## MODULE 4｜ディフェンス戦術大全

ここでは、具体的なディフェンス戦術を整理します。

個人ディフェンス、チームディフェンス、PNRカバレッジ、エリートスコアラーへの対応、トランジションディフェンス、ゾーン、プレスなどを扱います。

大事なのは、守り方をただ増やすことではなく、自チームの戦略や選手構成に合う守り方を選び、徹底できる形にすることです。

## MODULE 5｜対応戦術

ここでは、相手の戦術にどう対応するかを扱います。

相手がPNRをどう使ってくるのか。どの選手を起点にしているのか。ゾーンやプレスに対して、どのように崩し方を準備するのか。

戦術を知っているだけでは、試合中の問題には対応できません。MODULE 5では、相手の狙いを読み、自チームの解決策に変換するための考え方を整理します。

## MODULE 6｜戦略設計・システム構築

ここでは、本講座全体の核になる「設計」の話に入ります。

どんなバスケをしたいのか。そのシーズンのロスターで何を最大化するのか。どの戦略を立て、どのアクションやセットを選ぶのか。

My-vaultで整理している考え方でいえば、戦術はSystemの一要素でしかありません。Vision、Identity、Game Modelがあり、そのシーズンの戦力やロスターバランスに応じて戦略を定め、その戦略に合う戦術を選ぶ。この順番が崩れると、戦術はただの寄せ集めになります。

MODULE 6では、チームの勝ち筋から逆算して、オフェンス・ディフェンスのシステムをどう組み立てるかを扱います。

## MODULE 7｜試合分析

最後に、作った戦術やシステムをどう検証し、改善するかを扱います。

試合分析は、反省会のために行うものではありません。次の練習、次のゲームプラン、次の戦術修正につなげるために行うものです。

スタッツ、映像、現場での感覚をつなげながら、戦術を運用して終わりにせず、改善サイクルに乗せる方法を整理します。


## 読み方のポイント

この講座は、最初から順番に読むことで全体像がつかみやすくなるように構成しています。

ただし、読み方は1つではありません。

まず全体を一度通して読み、バスケットボール戦術を「原理から設計、分析までつながったもの」として捉えてください。そのうえで、自分のチームの課題に応じて必要なMODULEに戻る読み方がおすすめです。

たとえば、オフェンスが形だけになっているならMODULE 1とMODULE 6に戻る。具体的なアクション整理が必要ならMODULE 2を見る。ディフェンスの方針が曖昧ならMODULE 3とMODULE 4に戻る。試合中の対応やスカウティングに課題があるならMODULE 5とMODULE 7を見る。

このように、講座全体を「辞書」として使うのではなく、チームの問題を整理するための地図として使ってください。


## 読み進めるときの問い

各Lessonを読むときは、次の問いを持ちながら読み進めてください。

- この内容は、どのアドバンテージを作るためのものか
- 自チームのロスターで再現できるか
- どの選手の強みを活かす設計になっているか
- 相手にどのような判断を迫るものか
- 失敗するとしたら、どこで詰まりやすいか
- 練習では、どの部分を先に徹底すべきか
- 試合後には、何を見て修正すべきか

この問いを持つだけで、同じ戦術を見ても学び方が変わります。

単に「このセットは使えそうだ」と見るのではなく、「なぜこのセットが成立しているのか」「自チームで使うなら何を変えるべきか」「相手に対応されたら次に何を出すべきか」まで考えられるようになります。


## この講座で大事にする姿勢

本講座では、万能な正解を提示することを目的にしていません。

バスケットボールの戦術は、カテゴリー、ロスター、相手、ルール、練習時間、選手の理解度によって最適解が変わります。プロで有効な戦術が高校生にそのまま合うとは限りません。逆に、シンプルな原則でも、チームに深く浸透していれば大きな武器になります。

だからこそ大切なのは、戦術を「知っているかどうか」ではなく、なぜそれを選ぶのか、どのようにチームへ落とし込むのか、うまくいかなかったときに何を修正するのかです。

この講座では、具体的な戦術を扱いながらも、常にその背後にある原理、戦略、ロスター、運用、分析とのつながりを確認していきます。

次からは、いよいよ本編です。

MODULE 1では、まずオフェンス原論として、戦術を理解するための土台になるアドバンテージを整理します。具体的なセットやアクションを見る前に、「オフェンスは何を作り、何を使う営みなのか」を押さえることで、その後のオフェンス戦術、戦略設計、試合分析までがつながって見えるようになります。
