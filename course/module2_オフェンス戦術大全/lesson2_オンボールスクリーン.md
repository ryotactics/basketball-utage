# MODULE 2 - Lesson 2｜オンボールスクリーン


## 前のレッスンからのつながり

前のレッスンでは、オフェンス戦術の全体像と、各戦術がMODULE 1の共通言語（アドバンテージ／優位設計／プレーの4原則）にどう紐づくかを俯瞰しました。

このレッスンからは、各戦術の中身に入ります。まずは現代バスケの中核であるオンボールスクリーン——その代表格であるPNR（ピック&ロール） を見ていきます。


## このレッスンの目的

- オンボールスクリーンの原理を整理する
- PNRの基本構造とバリエーションを理解する
- オンボールスクリーンがどのアドバンテージを、どう作るのかを言語化する


## オンボールスクリーンの原理（MODULE 1との接続）

オンボールスクリーンは、ボールを持っている選手にスクリーンをかけるアクションです。

- 作り方：人を動かす × ボールを動かすの複合（スクリーナーが動き、ハンドラーがドリブル）
- 生まれるアドバンテージ：セパレーション／ミスマッチ／（副次的に）アウトナンバー
- 優位設計との接続：Self Strong（ハンドラーが強い）またはOpponent Weak（スイッチやDropに弱い相手を突く）の両方で使える
- プレーの4原則との関係：ディフェンスに「誰を守るか」「どう連携するか」という認知と予測を強制する

スクリーナーとハンドラーの2人が同時に関わるため、ディフェンス側は複雑な判断を迫られ、どこかに必ずズレが生まれます。


## 本文 — ユーザー執筆

このセクションでは、以下の観点からオンボールスクリーンを解説します。

### 1. PNR（ピック&ロール）の基本構造
- ハンドラー、スクリーナー、残り3人のスペーサーという5人構造で説明する
- ハンドラーの役割：スクリーンを使う、Rejectする、待つ、引きつける、パスを出す
- スクリーナーの役割：角度、タイミング、コンタクト、Roll／Pop／Slip／Short Rollの選択
- 残り3人の役割：Corner、Lift、Drift、Slot、Shake、Tagを罰する位置取り
- スクリーンは「2人の技術」ではなく、ヘルプをどこから出させるかまで含めた設計として扱う

### 2. PNRのバリエーション
- 角度の違い：High PNR、Side PNR、Step Up、Flat、Angle
- タイミングの違い：Drag、Early、Late Clock、Ram Screen後のPNR
- 人の組み合わせ：Guard-Big、Inverted、Spain、Ghost、77、Empty Side
- 出口の違い：Roll、Pop、Short Roll、Slip、Re-screen、Throwback
- 入り方の違い：Horns、21/Pistol、DHO、Chicago、ZipperからPNRへ入る形

### 3. ディフェンスのカバレッジごとの読み
- Drop：Pull-up、Floater、Short Roll、Pop、Bigを外に引き出す設計
- Switch：ミスマッチISO、Post、Seal、Slip、Ghost、Screen the Screener
- Hedge / Show：早いRoll Feed、Short Roll、Weak SideへのSkip、Rescreen
- Blitz：Release Point、Short Rollの判断、3rd Man、背後のカッティング
- ICE / Down：Reject、Step Up、Corner Lift、逆サイドへの展開
- カバレッジ名の暗記ではなく、「相手が何を消し、何を与えているか」で書く

### 4. 他アクションとの接続
- PNR → Pitch
- PNR → Rescreen
- PNR → DHO
- Ram → PNR
- Chicago → PNR
- PNR → Hammer / Lift / Drift
- PNRが守られた後に、次のアクションでアドバンテージを連鎖させる流れを入れる

### 5. コーチングで触れる観点
- ハンドラーに「見る順番」をどう教えるか
- スクリーナーに「スクリーン後の出口」をどう教えるか
- スペーサーに「何もしない時間」ではなく「ヘルプを罰する準備」として理解させる
- 練習では2on2だけでなく、Tag、Low Man、Weak Sideを入れた3on3〜5on5で扱う


## コーチが意識すべきこと

- 自チームのPNRはSelf StrongなのかOpponent Weakなのか、どちらで使っているか
- スクリーナーとハンドラーの2択をちゃんと用意できているか（スイッチされたらどうする／ヘッジされたらどうする）
- 残り3人のスペーシングまで含めて設計できているか

PNRは「2人の動き」ではなく「5人の設計」です。ここを言語化しきれていないと、同じアクションでも機能したりしなかったりします。
