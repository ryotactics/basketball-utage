# MODULE 6 - Lesson 1｜チームビジョン・コンセプト設計

---

## 前のモジュールからのつながり

MODULE 1〜5では、オフェンスとディフェンスの原論・各論・対応戦術を扱いました。

MODULE 6では、それらを**「チームとしてどう体系化し、運用するか」** という上位の設計に入ります。

MODULE 0（はじめに）で触れた「ビジョン → 戦略 → コンセプト → 戦術」という階層の、**一番上から順に設計する**プロセスを扱います。

---

## なぜ「戦術から始めてはいけないか」

多くのコーチは、「良い戦術を見つけた」「強いチームのセットを取り入れたい」という入口から戦術設計を始めます。

しかし、**戦術は目的の手段であり、目的のないまま手段を集めると機能しません。**

> 戦術は、戦略を実現するために、具体的にどう動くかといった手段・方法。
> 戦略のない戦術選択は、地図のない道歩きと同じ。

勝てるチームは「何をやるか」ではなく「**なぜそれで勝てるのか**」が明確です。その根拠を作るのが、このモジュールで扱う戦略・コンセプト設計です。

---

## 設計の階層

```
チームビジョン
  └ 「どんなバスケをしたいか」「どんなチームでありたいか」

戦略
  └ 「その戦力・ロスターで、どう勝つか」の全体方針

コンセプト
  └ 「オフェンス / ディフェンスの核となる考え方」

戦術
  └ 「コンセプトを実現するための具体的なアクション・セット」
```

この4層が**一貫してつながっているか**が、チームの軸がブレるかどうかを決めます。

---

## Vision / Game Model / Systemで分ける

チーム設計では、Vision、Game Model、Systemを分けて考えると混乱しにくくなります。

Visionは、チームがどこへ向かうかです。3〜5年後にどういうチームでありたいのか、何を大切にするのか、どんな成功指標で進捗を見るのかを示します。

Game Modelは、今季のバスケットをどう定義するかです。Paceを上げるのか、ハーフコートで詰めるのか、リムと3Pを優先するのか、OREBでポゼッション量を増やすのか。ここでは「美しいバスケ」ではなく、何を行動として最大化するかまで落とします。

Systemは、そのGame Modelを実際の戦術・原則・運用に変換する層です。Transition O、メインアクション、PnR Coverage、Transition D、練習設計、分析サイクルがここに入ります。

この3つを分けることで、「理念の話をしているのか」「今季の勝ち筋の話をしているのか」「明日の練習で何をするかの話をしているのか」が整理されます。

---

## チームビジョンとは

チームビジョンは、**「どんなバスケをするチームでありたいか」** という最上位の方針です。

これはコーチ個人の哲学・信念であり、戦力に依らない部分も含みます。

例：
- 「アグレッシブで攻撃的なバスケをする」
- 「5人全員が判断できるモーションバスケを目指す」
- 「ディフェンスで戦うチームを作る」

ビジョンが明確でないと、戦力に応じた戦略の調整も「何に向かって調整するか」が不明確になります。

---

## コンセプト設計とは

コンセプトは、ビジョンと戦力を踏まえた上で決める**オフェンス・ディフェンスの核となる考え方**です。

コンセプトは以下を規定します。

- **何を主武器とするか**（PnR主体 / オフボールスクリーン主体 / Pass & Cutting主体など）
- **誰を中心に置くか**（特定エースのISO / チーム全員が動くフロー型）
- **どのエリアでどう点を取るか**（リムアタック中心 / 3P中心 / ミドルレンジ活用）
- **ディフェンスの優先事項**（No Middle / Switch All / ゾーン混じりなど）

コンセプトが決まると、**選ぶべき戦術の範囲が自動的に絞られます。** これが「引き出しの多さ」より「コンセプトへの精度」が重要な理由です。

---

## コンセプトとロスターの整合性

コンセプトはロスター（選手構成）と整合していなければなりません。

| コンセプト | 必要なロスター条件 |
|---|---|
| PnR主体 | クリエイトできるガードが複数枚必要 |
| オフボールスクリーン主体 | スクリーニング精度の高いビッグ、正確なシューター |
| Pass & Cutting主体 | わずかなズレでもシュートを決められる選手の枚数 |
| MUM（ターゲットハンティング） | 周りをシューターで固め、カットを繰り返せる組み合わせ |

コンセプトと選手特性がズレていると、**どれだけ優れたセットを入れても機能しません。**

---

## コーチが意識すべきこと

- 自分のチームビジョンを**一言で説明できるか**
- ビジョン → 戦略 → コンセプト → 戦術が**一貫してつながっているか**
- 選手に「なぜこの戦術をやるのか」と聞かれたとき、**コンセプトから説明できるか**

コンセプトが選手に伝わっているチームは、試合中の判断がコーチの指示なしに成立します。コンセプトが伝わっていないチームは、いつもコーチが何かを叫ばなければなりません。

## まとめ

- 戦術から始めるのではなく、**ビジョン → 戦略 → コンセプト → 戦術**の順で設計する
- チームビジョンは「どんなバスケをしたいチームか」の最上位方針
- コンセプトはロスターと整合していることが前提
- コンセプトが浸透すると、**選手が試合中に自分で判断できるようになる**
