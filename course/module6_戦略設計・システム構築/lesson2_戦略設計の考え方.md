# MODULE 6 - Lesson 2｜戦略設計の考え方

---

## 前のレッスンからのつながり

前のレッスンでは、ビジョン → 戦略 → コンセプト → 戦術という4層の階層と、チームビジョン・コンセプトの設計を整理しました。

このレッスンでは、その中核となる**「戦略設計」**——ロスター診断から勝ち筋の定義、資源配分まで——を具体的に扱います。

---

## 戦略設計とは

戦略とは、**「目標を達成するための全体的な方針」** です。

バスケットボールにおける戦略設計は、以下を問い続けることです。

> 「この戦力・ロスターで、この競争環境の中で、どう勝つか」

戦略がないまま戦術を積み上げるチームは、うまくいかないとき「新しい戦術を増やす」「別のセットを試す」という局所的な修正に終始します。しかし本当に必要なのは、**「そもそも何で勝つか」を問い直すこと**です。

---

## 戦略設計のフロー

```
Step 1：ロスター診断
  └ 持っている選手の強み / 弱み / タイプの把握

Step 2：勝ち筋の定義
  └ このロスターで、相対的に優位を取れる「戦い方」は何か

Step 3：制約の確認
  └ 時間制約（シーズン、練習時間）
  └ 競争環境（対戦相手の傾向）
  └ 致命傷になる弱み（塞ぐべき弱点はどこか）

Step 4：資源配分の決定
  └ 何を使うか / 何を使わないか
  └ 練習時間をどこに集中させるか

Step 5：共通認識の設計
  └ 原則として定め、チーム全体に浸透させる
```

---

## 勝ち筋の定義

勝ち筋とは、**「この条件なら、相対的に優位を取れる構造」** です。

MODULE 1 Lesson 4の優位設計（Self Strong / Opponent Weak / Context / Information）と直結します。

- 個の突出した選手がいるなら → Self Strong（エースを活かした構造）
- 組織力・連動性が強みなら → Self Strong（チームとしてのモーション）
- 相手に特定の弱点があるなら → Opponent Weak（スカウティング優先）
- 接戦や終盤管理に強みがあるなら → Context（試合の状況をコントロール）

「自チームの最も期待値の高い戦い方」を決めることが、戦略設計の核心です。

---

## 目的関数と一次要因で考える

戦略を実務に落とすには、「勝ちたい」ではまだ抽象度が高すぎます。何を最大化し、何を最小化するのかを決める必要があります。

たとえば、Net Ratingを上げる、eFG%を上げる、TOV%を下げる、OREB%を上げる、相手のPaint Touchを減らす、相手のBad Shot Rateを上げる。こうした指標は、チームのGame Modelを数字へ翻訳するための道具です。

ただし、数字だけを追ってもチームは動きません。数字を動かす一次要因を決めます。Paceを上げてPossessionを増やすのか、0.5秒判断でCloseoutを攻めるのか、Transition Defenseで相手のEasy Basketを減らすのか。ここまで落とすと、戦略が練習内容に接続します。

戦略設計とは、抽象的な方針を、計測できる指標と日々の行動へ翻訳する作業です。

---

## 個別最適化の落とし穴

多くのコーチが陥るのは**「この選手を活かしたいからこの戦術をする」という個別最適化の積み重ね**です。

しかし、これは適用範囲が狭くなります——その選手がいるときだけ機能するオフェンスになってしまいます。

正しい順序は逆です。

```
❌ セット・アクション → 選手への当てはめ
✅ ロスター診断 → 戦略 → コンセプト → セット・アクション → 個別最適化
```

個別最適化（特定選手のためのオプション・コール）は、**全体設計の末端で行うもの**です。

---

## コア／サブパッケージの設計

戦略設計の実務では、**コアパッケージ**と**サブパッケージ**を区別して持つことが重要です。

| 種類 | 定義 | 役割 |
|---|---|---|
| **コアパッケージ** | コンセプトのメインでやること。再現性と精度を追求する | 試合の軸。ここで勝ちにいく |
| **サブパッケージ** | コンセプトとは少しズレた手札。イージーバスケット創出 | リズムを作り、コアを機能させやすくする |

強いチームはコンセプトに沿いながらも、多様な手札（サブパッケージ）を持っています。コアだけに固執するとオフェンスが重くなり、サブだけになるとコンセプトが失われます。

---

## コアに傾倒しすぎない

コアパッケージはチームの軸ですが、軸に傾倒しすぎるとオフェンスは重くなります。

たとえば、MUMやTarget Huntingをコンセプトにしているチームが、毎回同じようにターゲットを探すだけになると、ボールが止まり、周囲の選手のリズムが消えます。オフボールスクリーンとFlowを軸にしているチームでも、全員が流れ続けるだけで、特定の選手が強みを出す場面がなければ、勝負どころで得点が止まります。

だからこそ、コアから大きく外れない範囲で、少し違う手札を持つことが重要です。速いテンポが軸のチームでも、落ち着かせる時間帯にはDelayやElbowを使う。オフボール中心のチームでも、特定選手にはMUM ISOを許可する。Pick主体のチームでも、相手のCoverageによってGhostやDHOを混ぜる。

サブパッケージは、コアを否定するものではありません。むしろ、コアを通しやすくするための逃げ道であり、相手を慣れさせないための揺さぶりです。

---

## 共通認識としての原則

戦略を形にするためには、**チーム全体の共通認識として「原則」を制定する**必要があります。

共通認識として定めるべき主な原則例：

- Transition Rule（帰陣・速攻のルール）
- Early Offense のスタート条件
- クローズアウトを受けたときの3択判断（Shoot / Drive / Pass）
- キックアウトを受け取ったときの判断
- Off Ball Cutting / React / Spot Up の優先順位
- スクリーンの使い方と角度
- OREBを取った後の判断

原則は「なんとなく」ではなく、**コンセプト・戦略に基づいて設計されたもの**である必要があります。でないと、練習で繰り返しても「なぜそうするのか」が選手に伝わりません。

---

## コーチが意識すべきこと

- 自チームの**勝ち筋を一言で説明できるか**
- 練習時間の配分は、**コア / サブパッケージの比率**と合っているか
- 設定した原則は**戦略・コンセプトから逆算されたものか**

戦略設計は「最初に一度決めれば終わり」ではありません。シーズンを通じた選手の成長、対戦相手の変化に応じて**継続的に見直す**ものです。

## まとめ

- 戦略設計のフローは**ロスター診断 → 勝ち筋定義 → 制約確認 → 資源配分 → 原則制定**
- 勝ち筋は**Self Strong / Opponent Weak / Context / Information**から根拠を選ぶ
- 個別最適化は全体設計の**末端**で行う
- **コア / サブパッケージ**を区別して持つことで、試合中の柔軟性が増す
