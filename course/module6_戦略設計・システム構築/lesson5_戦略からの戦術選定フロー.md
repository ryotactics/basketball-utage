# MODULE 6 - Lesson 5｜戦略からの戦術選定フロー


## 前のレッスンからのつながり

前のLesson 3・4でオフェンス / ディフェンスのシステム構築を整理しました。

このレッスンでは、「戦略 → 戦術」の選定フローを実務的に整理します。

「なぜこのアクションを選ぶのか」を言語化できるコーチは、試合中の修正も、選手への説明も、新しい戦術の導入判断も的確になります。


## 戦術選定の問い

アクションやセットを選ぶときに自分に問うべき問いは、以下の3つです。

### 1. このアクションは自チームのコンセプトに合っているか
   → コンセプトとズレたアクションを入れると「引き出しの増加」ではなく「コンセプトの希薄化」になる

### 2. このアクションは自チームのロスターで遂行できるか
   → どんなに優れた戦術でも、実行できる選手がいなければ機能しない

### 3. このアクションはどの優位設計に紐づいているか
   → Self Strong / Opponent Weak / Context / Information のどれで使うか

この3問に答えられないアクションは、導入しないほうが良いことが多いです。


## 選定フローの全体像


1. ロスター診断：持っている選手の強み、弱み、タイプを整理する。

2. 勝ち筋定義：どの優位設計で戦うのかを決める。

3. コンセプト決定：主武器や守り方の核を定める。

4. Transition / Early Offense の設計：試合の入口でどのように優位を作るかを決める。

5. Main Action（コアパッケージ）の選定：チームの軸になるアクションを決める。

6. カウンター / リアクトの設計：相手の対応に対して何を読むか、何を返すかを決める。

7. セット（ATO / SLOB / スターターなど）の選定：場面別に使うセットを選ぶ。

8. オプションの個別最適化：特定選手の強みを活かす細部を加える。

9. 原則の制定と練習への落とし込み：チーム全体で共有する判断基準に変換する。

この順序を守ることで、「なぜこの戦術か」がすべて上位の設計から説明できる状態になります。

このフローは、単なる整理表ではありません。現場で迷ったときに戻る判断順です。

たとえば「新しいセットを入れたい」と感じたとき、最初に問うのはセット名ではなく、ロスターと勝ち筋です。このロスターは何で優位を取れるのか。その勝ち筋に対して、今足りないのはTransitionの交通整理なのか、Earlyの入口なのか、Main Actionの精度なのか、Counterの不足なのか。ここを飛ばしてセットを追加すると、プレイブックは厚くなっても、チームの判断は遅くなります。

戦術選定は、下から積み上げる作業ではなく、上位設計から絞り込む作業です。ロスター、勝ち筋、コンセプト、制約、競争環境を見たうえで、必要なアクションだけを選び、必要なオプションだけを尖らせます。


## 相手カバレッジによる選定

試合の中では、相手のカバレッジに応じてアクションの頻度・選択を変えます。

例：
- 相手がSwitchを多用するなら → Switchを誘発するPNRの頻度を上げ、Switch後のMUMをコアにする
- 相手がDropを多用するなら → プルアップジャンパーが打てるハンドラーのPNRを増やす
- 相手がTop Lockしてくるなら → Reject系のカウンターや別のUser（別選手）への切り替え

これは戦術の変更ではなく、コンセプトの中でアクションの頻度と優先順位を変えることです。コンセプトは維持しながら、相手への対応は柔軟に変えます。

ここで重要なのは、相手への対応を「別物を入れること」と考えすぎないことです。普段使っているアクションの頻度を上げる、同じアクションの起点を変える、配置をEmptyにする、Screenをかける選手を変える、Counterの優先順位を変える。こうした小さい変更でも、相手の守り方に対して十分なアジャストになる場合があります。

相手が対応してきたときは、まず既存のコアパッケージ内で解決できるかを見ます。それで足りなければサブパッケージを使い、さらに必要なら相手専用のDesign Playを使う。この順番にすると、試合中の修正がチームの共通認識から外れにくくなります。


## 「削る」判断の重要性

戦術設計において「何を追加するか」と同じくらい、「何を削るか」が重要です。

選択肢が多すぎると：
- 選手の判断が遅くなる（1/4秒の迷いでアドバンテージが消える）
- 練習時間が分散して精度が上がらない
- コンセプトが希薄になる

少ない引き出しを高精度で実行できるチームのほうが、多くの引き出しを持つが雑なチームより強い——これは現場でのあるある事例です。

シーズン中に戦術を「追加したい」衝動が出たとき、まず問うのは「既存の引き出しの精度は十分か」です。


手札は「数」ではなく「種類」で持つ

オフェンスの手札というと、セット数やプレーコール数を増やすことを想像しがちです。しかし、実際に重要なのは数ではなく、手札の種類と性質の幅です。

似た性質のセットを10個持っていても、相手がその系統を消してきたらまとめて重くなります。反対に、少ない数でも、On Ball Screen、Off Ball Screen、Post / Elbow Hub、DHO、MUM、Transition、Decoyなど、異なる負荷を相手にかけられる手札があれば、試合中にリズムを変えられます。

手札を見るときは、以下の観点で分類します。

- どのアドバンテージを作る手札か：セパレーション / ミスマッチ / アウトナンバー
- 誰を活かす手札か：Handler / Shooter / Big / Wing / Screener
- どのテンポで使う手札か：Transition / Early / Half Court / ATO
- どの守り方への返しになるか：Switch / Drop / Show / Top Lock / Denial
- その手札はコアなのか、サブなのか、相手専用なのか

この分類ができていないと、プレイブックは厚くなっても、試合中の選択は増えません。逆に、手札の種類が整理されていると、「今日はDrop対策の手札を増やす」「相手がTop LockしてくるからOff Ballの出口を変える」「得点が重いからサブパッケージでイージーを作る」といった判断がしやすくなります。


カウンターを階層で持つ

相手に対応されたとき、すぐに別セットを入れる必要はありません。カウンターには階層があります。


### 1. エントリーが阻止されたときのカウンター
2. Action / Setそのものに対応されたときのカウンター
### 3. Kill the Coverage（その守り方自体をさせない）
### 4. 構造の弱点を突くカウンター
### 5. 別コンセプトで攻める

この順番を持っていると、試合中の修正が整理されます。

たとえば、PNRにShowされて困っているとします。まずはHandlerのRejectやScreenerの早いReleaseで直接対応できるかを見る。次にRamなどでShowのタイミングを遅らせる。さらにSpacingや配置を変えてTagを出しにくくする。それでも重いなら、Off Ball ScreenやPost Hubなど、別コンセプトへ移る。

重要なのは、カウンターを「公式」として覚えるだけで終わらせないことです。実際の試合では、Coverageの位置、Helpの寄り方、Tagの深さ、相手のコミュニケーションの癖が少しずつ違います。公式を知った上で、相手の微妙な変化に対して、配置・テンポ・Option・Counterをどう変えるか。ここが実戦のアプローチです。


## 戦術はロスター要件とセットで選ぶ

戦術を選ぶときは、「その戦術が良いか」だけでなく、「その戦術に必要なロスター要件を満たしているか」を確認します。

PNRを軸にするなら、Handlerの認知帯域、ScreenerのRoll / Pop / Short Roll能力、WeaksideのSpot UpとCut、Bigの接触圧が必要になります。5-Out Motionを軸にするなら、全員の判断速度、パス能力、外のSpacing、Off Ball Reactが問われます。Switch Defenseを軸にするなら、サイズ、機動力、Mismatch解消、DREBが必要です。

ロスター要件を満たしていない戦術は、形としては実行できても、試合では機能しません。逆に、ロスターの強みと噛み合った戦術は、シンプルでも強い武器になります。

戦術選定は、戦術リストから選ぶ作業ではなく、ロスターとGame Modelの間に橋を架ける作業です。

## コアパッケージとサブパッケージ

オフェンスを設計するときは、チームの中心となる手札と、流れを変えるための手札を分けます。

コアパッケージは、コンセプトに沿ったメインの攻め方です。練習時間を最も多く使い、選手が迷わず戻れる基準になります。PNR主体、Off Ball Screen主体、MUM主体、Post / Elbow Hub主体など、チームの勝ち筋に直結する部分です。

サブパッケージは、コアとは少し違うが、コアを通しやすくするための手札です。相手がコアに慣れたとき、得点が重くなったとき、テンポを変えたいとき、特定ラインナップで違う強みを使いたいときに使います。重要なのは、サブパッケージを完全に別のバスケにしないことです。できるだけ同じ原則、同じ配置、同じ選手の役割から派生させると、選手の実行負荷を抑えながら幅を作れます。

コンセプトに傾倒しすぎると、相手に慣れられたときにオフェンスが重くなります。一方で、手札を増やしすぎると、チームの軸がぼやけます。コアを明確にし、サブを必要最小限に絞ることが、現場で機能する戦術選定です。

## コール選定の実務

戦術選定は、試合前のプレイブック作成だけでは終わりません。試合中に、コーチやガードがどのコールを選ぶかまで含めて設計する必要があります。

コールを選ぶときの判断軸は、次の通りです。

- 相手の出方を確認したいのか
- 自分たちの強みを押し付けたいのか
- 相手の弱みを突きたいのか
- 自分たちの強みで相手の弱みを突きたいのか
- 3Pが欲しいのか、確実な2点が欲しいのか
- テンポを上げたいのか、落ち着かせたいのか
- 相手のカウンターに対してWrinkleを出すのか
- いま効いているプレーをもう一度使うのか

この判断軸がないと、コールは「思いつき」になります。反対に、判断軸が共有されていると、ガードが自分でコールを選ぶ場合でも、コーチの意図とズレにくくなります。

また、同じコールでも配置やラインナップによって選ぶOptionは変わります。シューターがHandlerになるならGetへ移る。BigをDunkerに置きたいならRipやScreen the Screenerへ繋げる。Handlerが強い時間帯はコールを絞り、Off Ballで活きる選手が多い時間帯はExitやStaggerを増やす。コールは「固定された型」ではなく、ラインナップと目的に応じて運用するものです。


## スカウティングと戦術選定

相手スカウティングに基づいて、試合ごとに戦術選定を調整します。

スカウティングで判明すること / 戦術選定への反映
相手のPNRカバレッジ傾向 / カウンターの優先順位を変える
相手の弱いマッチアップ / そのマッチアップを引き出すセットを追加
相手が苦手な形態（ゾーン / プレスなど） / サブパッケージの活用頻度を上げる
相手エースのハンドリング力 / プレスやトラップの発動条件を緩める

スカウティングはOpponent Weakの情報源です。試合ごとにコアの上にスカウティングのレイヤーを追加するイメージで設計します。


## コーチが意識すべきこと

- 導入しているすべてのアクション・セットに「なぜこれか」の根拠があるか
- シーズン中に「削る」判断を定期的にしているか
- 手札を数ではなく、種類・目的・テンポ・相手への負荷で整理できているか
- 相手へのカウンターを、直接対応だけでなく階層で持てているか
- スカウティングの情報を戦術選定に反映できる仕組みがあるか（個人かスタッフかを含めて）

## まとめ

- 戦術選定の問いはコンセプトに合うか / ロスターで遂行できるか / どの優位設計かの3問
- 選定フローはロスター → 勝ち筋 → コンセプト → メインアクション → セット → オプション → 原則
- 相手カバレッジへの対応はコンセプトを変えずに頻度・優先順位を変える
- 「追加」より「削る」判断がシステムの精度を上げる
- 手札は数ではなく種類と性質の幅で持つ
- カウンターはエントリー / Action / Coverage / 構造 / 別コンセプトの階層で整理する
