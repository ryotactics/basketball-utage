# MODULE 6 - Lesson 6｜システムの運用とメンテナンス

---

## 前のレッスンからのつながり

Lesson 1〜5でビジョン・戦略・コンセプト・システム構築・戦術選定フローを整理してきました。

このレッスンはMODULE 6の締めくくりです。設計したシステムを**実際に機能させ、シーズンを通じて更新し続ける**——運用とメンテナンスを扱います。

「設計すること」と「機能させること」は別のスキルです。

---

## 運用のサイクル

システムは以下のサイクルで運用します。

```
設計（プレシーズン）
  ↓
練習での浸透（シーズン序盤）
  ↓
試合での実行 → 観察
  ↓
分析（機能しているか / していないか）
  ↓
修正・最適化
  ↓
再び試合へ
```

このサイクルを意識的に回しているチームと、「試合 → 感覚的な修正 → 試合」を繰り返しているチームでは、シーズン後半の完成度が大きく変わります。

---

## 浸透させるための練習設計

設計したシステムが機能するためには、**選手がコンセプトと原則を体に染み込ませる練習**が必要です。

### 段階的な浸透の設計

| ステップ | 内容 |
|---|---|
| **概念の共有** | なぜこのシステムか、何を目的とするかをコーチが言語化して伝える |
| **パーツ練習** | 個別のアクション・原則を切り出してドリルで反復 |
| **シチュエーション練習** | 実際の試合局面に近い条件で設計を試す（例：PnR → カバレッジへの対応） |
| **スクリメージでの確認** | チーム全体で連動させながら精度を確認 |
| **試合での観察** | 練習でできていることが試合でも出せているかをチェック |

---

## 「できていないこと」の分析

試合でうまくいかないとき、コーチは2つを区別する必要があります。

```
戦術の問題（設計が悪い）
  └ コンセプトや戦術選定の見直しが必要

実行の問題（選手の精度・理解が足りない）
  └ 練習設計・浸透の見直しが必要
```

この区別ができないと、**実行の問題なのに戦術を変える**（問題が残る）、または**設計の問題なのに練習を増やす**（無意味な反復）という誤りが起きます。

---

## Keep / Try / Dropで更新する

システム運用では、すべてを毎回ゼロから見直す必要はありません。継続するもの、試すもの、捨てるものを分けると整理しやすくなります。

Keepは、機能しているため継続するものです。結果が悪い試合でも、構造として良かったものは残します。

Tryは、次の一定期間で試すものです。新しいCoverage、ラインナップ、セット、練習ルールなど、検証対象として扱います。

Dropは、今のロスターやGame Modelに合わないもの、練習時間に対して効果が薄いものです。削る判断ができないと、システムは肥大化して実行精度が落ちます。

この3分類を週次や試合後レビューで使うと、感情的な反省ではなく、運用としての改善サイクルになります。

---

## シーズンを通じたシステム更新

システムはシーズンを通じて**変化させながら使う**ものです。

### 主な更新のトリガー

- **相手にバレてきたとき** → Fake Actionを増やす / セットのバリエーションを加える
- **選手の成長によって** → オプションの範囲を広げる / 任せる判断を増やす
- **ロスターの変化によって** → コンセプトや主アクションを調整する
- **対戦相手に応じて** → スカウティングに基づいたサブパッケージの調整

---

## シーズン前・中・後の視点

| タイミング | 主なアクション |
|---|---|
| **プレシーズン** | コンセプト設計、原則の言語化、基本システムの構築と浸透 |
| **シーズン序盤** | システムの精度向上、課題の特定、スカウティング体制の確立 |
| **シーズン中盤** | 修正・最適化。情報優位のFake Action追加。ロスター変化への対応 |
| **シーズン終盤** | プレーオフ / 重要戦向けのコアの研ぎ澄まし。削ることを優先 |
| **オフシーズン** | 今シーズンの振り返りと次シーズンのビジョン・戦略再設計 |

---

## 選手の判断を育てる

優れたシステムの最終目標は、**選手が自分で判断できるようになること**です。

コーチが指示を出さなくても、コンセプトと原則に基づいて選手が動ける状態——これが「チームとしての完成」です。

ただし、判断を選手に任せすぎることの危険もあります。IQが高い選手でも、明確な基準がなければ判断が個人ごとにバラバラになり、チームの軸がブレます。

**コーチが設計した原則の中で、選手が自由に動ける**——この構造が理想です。

## まとめ

- システムの運用は**設計 → 浸透 → 実行 → 分析 → 修正**のサイクルで回す

- 最終目標は**選手が原則の中で自分で判断できるチーム**を作ること
