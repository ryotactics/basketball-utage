# MODULE 6 - Lesson 6｜システムの運用とメンテナンス


## 前のレッスンからのつながり

Lesson 1〜5でビジョン・コンセプト・戦略・システム構築・戦術選定フローを整理してきました。

このレッスンはMODULE 6の締めくくりです。設計したシステムを実際に機能させ、シーズンを通じて更新し続ける——運用とメンテナンスを扱います。

「設計すること」と「機能させること」は別のスキルです。


## 運用のサイクル

システムは以下のサイクルで運用します。


1. 設計（プレシーズン）：チームのコンセプト、戦略、原則を定める。

2. 練習での浸透（シーズン序盤）：選手が判断基準として使える状態まで落とし込む。

3. 試合での実行と観察：設計した内容が実際の試合で機能しているかを見る。

4. 分析：機能している部分と、機能していない部分を分ける。

5. 修正・最適化：戦術、原則、練習内容、役割配分を調整する。

6. 再び試合へ：修正した内容を次の試合で検証する。

このサイクルを意識的に回しているチームと、「試合 → 感覚的な修正 → 試合」を繰り返しているチームでは、シーズン後半の完成度が大きく変わります。


## 浸透させるための練習設計

設計したシステムが機能するためには、選手がコンセプトと原則を体に染み込ませる練習が必要です。

## 段階的な浸透の設計

| ステップ | 内容 |
| --- | --- |
| 概念の共有 | なぜこのシステムか、何を目的とするかをコーチが言語化して伝える |
| パーツ練習 | 個別のアクション・原則を切り出してドリルで反復 |
| シチュエーション練習 | 実際の試合局面に近い条件で設計を試す（例：PNR → カバレッジへの対応） |
| スクリメージでの確認 | チーム全体で連動させながら精度を確認 |
| 試合での観察 | 練習でできていることが試合でも出せているかをチェック |


段階に応じた介入

システムを浸透させるとき、コーチの介入は選手やチームの習得段階によって変わります。

最初からすべてを選手の判断に任せても、選手は何を基準に判断すればいいかわかりません。逆に、熟達している選手にいつまでも細かい指示を出し続けると、自律的な判断が育ちません。

| 段階 | 目的 | コーチの介入 |
| --- | --- | --- |
| 初期 | 基本型と内部モデルを作る | 原理、基本型、成功例を明確に提示する |
| 中間 | 原理を状況に適用する | 問いかけ、制約つきゲーム、判断基準の確認を使う |
| 熟達 | 即時適応と自律化 | 原則の再確認と環境設計に留め、選手同士の修正を促す |

同じ選手でも、コンセプトによって段階は変わります。PNRは熟達していても、新しく入れたSpain PNRの読みは初期段階かもしれません。コーチは「選手 × コンセプト」で、どの段階にいるかを見極める必要があります。

練習設計の最終目的は、コーチが常に答えを出すことではなく、選手が原則に基づいて自分たちで修正できる状態へ進めることです。


## 「できていないこと」の分析

試合でうまくいかないとき、コーチは2つを区別する必要があります。


戦術の問題（設計が悪い）
  └ コンセプトや戦術選定の見直しが必要

実行の問題（選手の精度・理解が足りない）
  └ 練習設計・浸透の見直しが必要

この区別ができないと、実行の問題なのに戦術を変える（問題が残る）、または設計の問題なのに練習を増やす（無意味な反復）という誤りが起きます。


## Keep / Try / Dropで更新する

システム運用では、すべてを毎回ゼロから見直す必要はありません。継続するもの、試すもの、捨てるものを分けると整理しやすくなります。

Keepは、機能しているため継続するものです。結果が悪い試合でも、構造として良かったものは残します。

Tryは、次の一定期間で試すものです。新しいCoverage、ラインナップ、セット、練習ルールなど、検証対象として扱います。

Dropは、今のロスターやGame Modelに合わないもの、練習時間に対して効果が薄いものです。削る判断ができないと、システムは肥大化して実行精度が落ちます。

この3分類を週次や試合後レビューで使うと、感情的な反省ではなく、運用としての改善サイクルになります。


## シーズンを通じたシステム更新

システムはシーズンを通じて変化させながら使うものです。

### 主な更新のトリガー

- 相手にバレてきたとき → Fake Actionを増やす / セットのバリエーションを加える
- 選手の成長によって → オプションの範囲を広げる / 任せる判断を増やす
- ロスターの変化によって → コンセプトや主アクションを調整する
- 対戦相手に応じて → スカウティングに基づいたサブパッケージの調整


## シーズン前・中・後の視点

タイミング / 主なアクション
プレシーズン / コンセプト設計、原則の言語化、基本システムの構築と浸透
シーズン序盤 / システムの精度向上、課題の特定、スカウティング体制の確立
シーズン中盤 / 修正・最適化。情報優位のFake Action追加。ロスター変化への対応
シーズン終盤 / プレーオフ / 重要戦向けのコアの研ぎ澄まし。削ることを優先
オフシーズン / 今シーズンの振り返りと次シーズンのビジョン・戦略再設計


## 選手の判断を育てる

優れたシステムの最終目標は、選手が自分で判断できるようになることです。

コーチが指示を出さなくても、コンセプトと原則に基づいて選手が動ける状態——これが「チームとしての完成」です。

ただし、判断を選手に任せすぎることの危険もあります。IQが高い選手でも、明確な基準がなければ判断が個人ごとにバラバラになり、チームの軸がブレます。

コーチが設計した原則の中で、選手が自由に動ける——この構造が理想です。
