# MODULE 7 - Lesson 1｜試合分析の全体像


## 前のモジュールからのつながり

MODULE 6では、ビジョン・戦略・システムの設計と運用サイクルを扱いました。

その運用サイクルの中で欠かせないのが試合分析です。設計したシステムが機能しているかを確認し、次の改善につなげる——この「観察と改善」のループが、チームの成長を駆動します。

MODULE 7は、この講座の最後のモジュールです。分析の視点と方法を体系化し、現場で使える形に整えます。


## 試合分析の目的

試合分析の目的は、「何が起きたかを正確に把握し、次にどう変えるかを決めること」 です。

「なんか守れてなかった」「オフェンスが重かった」という印象論ではなく、構造的に何が問題だったのかを特定することが目的です。

試合分析の結果は、以下に活かします。

- 次の試合への準備（スカウティング / 戦術選定）
- チームへのフィードバック（選手へのコーチング）
- システムの修正・更新（MODULE 6の運用サイクルへの入力）


## 試合分析の3つのレイヤー

試合分析は、以下の3レイヤーで構成されます。


### 1. スタッツ分析（数字で全体像を把握する）
  └ 得点・失点・効率の定量的な把握
  └ 問題のエリアを絞り込む

### 2. 映像分析（現象の原因を見る）
  └ なぜその数字になったのかを映像で確認
  └ 戦術的な問題 / 判断の問題 / 実行の問題を区別する

### 3. 改善サイクルへの接続
  └ 分析結果を次の練習・試合準備・システム更新につなげる

この3つのレイヤーは独立ではなく、スタッツで「何が」を掴み、映像で「なぜ」を確認し、サイクルで「次に何を」を決める流れで使います。


## 分析の2つの対象

試合分析には自チーム分析と相手チーム分析（スカウティング） の2種類があります。

対象 / 目的
自チーム分析 / 設計したシステムが機能しているかの確認、課題の特定
相手スカウティング / 相手の強み / 弱み / 傾向の把握、Opponent Weak の発見

どちらも同じ分析ツール（スタッツ / 映像）を使いますが、問いの方向が逆です。

自チーム分析は「自分たちに何が足りないか」を見る。相手スカウティングは「相手に何を狙えるか」を見る。


## 分析は「答え探し」ではなく「問いの整理」

試合分析で最初にやるべきことは、映像を細かく見ることではありません。まず、何を知りたいのかを決めることです。

「なぜ負けたのか」という問いは広すぎます。これでは、気になった場面を拾うだけになり、分析が感想に近づきます。

問いは、できるだけ具体化します。

- なぜ3Qの立ち上がりに失点が増えたのか
- Dropに対してPull Upを打たれたのか、Rollを使われたのか
- Transitionで走られた原因は、Shot Selectionか、Safetyの配置か
- Offenseが止まった原因は、Spacingか、相手Coverageか、判断速度か

問いが具体的になると、見るべき映像、取るべきスタッツ、選手へ伝える内容が絞られます。分析の質は、問いの質で大きく変わります。


結果から設計階層へ戻す

試合分析で大事なのは、結果を見て終わらないことです。

たとえば「3Pを多く打たれた」という結果があったとします。そこで止まると、「もっとCloseoutしよう」という表面的な修正になります。しかし実際には、原因は複数考えられます。

- そもそもゲームプランとして、そのシューターを許容していたのか
- Helpの出どころが間違っていたのか
- PNR Coverageの選択が相手と合っていなかったのか
- ローテーションの最後が間に合っていなかったのか
- オフェンスのShot Selectionが悪く、Transitionから打たれていたのか
- ラインナップ上、Closeout距離が長くなりすぎていたのか

このように、結果は必ず上位の設計階層へ戻して確認します。


結果
  ↓
現象
  ↓
原因
  ↓
戦術選定
  ↓
コンセプト
  ↓
ロスター / Game Model

この戻し方ができると、分析が「反省」ではなく「設計修正」になります。逆に、この階層を持たない分析は、毎回その場の感想や根性論に戻りやすくなります。


## 分析をチームに活かすために

分析の質が高くても、チームに伝わらなければ意味がありません。

- スタッツだけでは「なぜ」が伝わりにくい
- 映像だけでは「全体像」が見えにくい
- 両方を組み合わせて、「数字 + 映像の具体例」で伝えるのが最も効果的

また、コーチが分析するだけでなく、選手が自分で分析する文化を作ることで、チームとしての学習速度が上がります。


## コーチが意識すべきこと

- 試合分析を習慣として組み込んでいるか（やったり / やらなかったりでは機能しない）
- 結果を、戦術・コンセプト・ロスターまで遡って確認できているか
- 分析の結果が次の練習や試合準備に具体的に反映されているか
- 分析にかける時間と、見る対象の優先順位を決めているか

分析に時間をかけすぎても、練習や準備の時間が削られます。限られた時間で最も重要な情報を取る設計が必要です。


## 次のレッスンへ

次のレッスンでは、試合分析の第1レイヤー——スタッツ分析の基本を扱います。どの数字を見るべきか、その数字が何を意味するかを整理します。


## まとめ

- 試合分析の目的は「何が起きたか」を正確に把握し、「次に何を変えるか」を決めること
- 分析はスタッツ（全体像）→ 映像（原因）→ 改善サイクル（次への接続） の3レイヤー
- 自チーム分析 / 相手スカウティングの2方向がある
- 結果は必ず、現象 → 原因 → 戦術選定 → コンセプト → ロスター / Game Modelへ戻して確認する
- 分析の価値はチームへの伝え方と改善サイクルへの接続で決まる
